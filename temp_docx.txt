



전체 흐름은 ① 초기화 및 사전 로드 → ② 메인 루프(동기화 감시) → ③ 데이터 검증 → ④ 화면 갱신 및 비동기 분산 로드 → ⑤ 시스템 감시 및 예외 처리 순서입니다.


Phase 1. 초기화 및 리소스 준비 (Initialization &amp; Setup)
앱 실행 시 단 한 번 수행되며, 현장 환경 설정과 미디어 자료구조 생성, 그리고 첫 버퍼링을 준비합니다.

설정 외부화 (Config Load): * 제공된 
CSVReader.cs
를 통해 외부 Settings.txt 파싱. 

Key(A열), Value(B열), Comment(C열)의 3열 구조 사용 (예: Display1_Resolution, 1920x1080, 서브 모니터 해상도).


파싱된
 해상도 문자열은 Split('x')로 분리하여 
Display.displays
[#
].Activate
(width, height, 60)에 즉시 적용 (유니티 
재빌드
 없이 현장 수정 가능).


크로스페이드
 딜레이 등 로컬 환경 변수 확보.

미디어 자동 파싱 및 자료구조 생성: * 
StreamingAssets
/Media# 폴더 탐색 → 파일명 
정규식
(##_#
#.*
) 분석. 

모니터 번호를 Key로 하는 Dictionary&lt;int, List&lt;
MediaData
&gt;&gt; 형태로 플레이리스트 구축.


파싱된
 데이터를 기반으로 Index(순서), Type(비디오/이미지), Duration(재생 시간)을 담은 커스텀 클래스 
MediaData

파싱 시 찾은 파일의 절대 경로를

캐싱하여
, 이후 재생 시 폴더 
재검색

로드하도록

초기 핑퐁 
버퍼링
 세팅 (Initial Buffering): 더블 버퍼링용 2개의 
AVPro
 Video 캔버스(버퍼 A, 버퍼 B)를 Screen Space - Camera (Order: -1)로 배치. 

앱 실행 즉시 버퍼 A에 '0번 미디어(
캐싱된
 
FilePath
)'
를
 
로드하여
 재생(Play) 시작.

동시에 보이지 않는 버퍼 B에 '1번 미디어'의 첫 프레임을 미리 디코딩(Prepare) 해두고 대기.

Phase 2. 메인 루프 (Sync Update Loop)
매 프레임 미디어 재생 상태와 0번 메인 모니터(Master) 기준의 동기화를 감시합니다.

마스터(Master) 기준 감시: 0번 메인 화면(Media0)의 재생 시간 및 파일 종료(EOF)를 지속 체크하여 전체 시스템의 '시계' 역할 수행.

서브 대기 상태: 서브 화면들(Media#)은 마스터 영상이 끝날 때까지 자체 영상 또는 이미지를 Loop.

무반응(Freeze) 체크: 
VideoWatchdog.cs
가 프레임 정지 여부를 판별하여 True일 경우 즉시 Phase 5로 진입.

Phase 3. 데이터 처리 및 검증 (Media Processing &amp; Validation)
재생 완료 시점과 다음 미디어로 넘어갈 준비 상태를 판단합니다.

재생 완료 검증: 마스터 화면이 
정규식
 파싱 시간(예: 10초)에 도달했거나 실제 영상 끝(End of Stream)에 도달했는지 확인.

글로벌 동기화: 마스터 화면의 재생이 끝났더라도, 보이지 않는 뒤쪽 캔버스(버퍼 B)에 모든 모니터의 다음 순서 미디어가 Prepare 완료되었는지 확인. 모두 Ready 상태일 때만 전환 승인.

[지연 발생 시] 한 곳이라도 준비가 되지 않았다면, 전체 화면 전환을 취소하고 로그 기록 후 Phase 5(리셋)로 이관한다.

동기화 타이밍 검증: 갱신 이벤트 발생 시, 
슬레이브
 화면의 현재 재생 위치와 무관하게 즉시 다음 미디어 인덱스로 갱신.

Phase 4. 화면 갱신 및 비동기 분산 로드 (Seamless Update &amp; Async Load)
검은 화면 없이 화면을 전환하고, 다음 미디어를 부하 없이 
로드하며
 딥 메모리 해제를 수행합니다.


크로스페이드
 전환: 준비된 뒤쪽 버퍼의 투명도(Alpha)를 1로, 기존 버퍼를 0으로 
교차시킴
. Settings.txt에서 가져온 딜레이만큼 전환, 깜빡임 차단.

부하 분산: 5~10개 이상의 모니터를 동시에 Prepare를 하면 재생 중인 동영상이 끊김 현상이 일어날 수 있습니다. 모든 화면을 동시에 
로드하지
 말고 0.1 ~ 0.3초 정도의간격을 두고 순차적으로 로드를 시작.

메모리 해제 및 다음 미디어 준비: 화면에서 완전히 사라진 버퍼의 데이터(이미지는 텍스처 Destroy, 비디오는 컴포넌트 
CloseVideo
)
를
 즉시 파기. 

비워진 이전 버퍼에 곧바로 '다음 순서(+2) 미디어'
를
 
캐싱된
 
FilePath
를 통해 Prepare 시켜 무한 순환 구조 완성.

주기적인 
가비지
 컬렉션(
Resources.UnloadUnusedAssets
) 호출로 장시간 구동 시 RAM 용량 폭주 차단.

Phase 5. 시스템 감시 및 예외 처리 (Exception Handling &amp; Crash Recovery)
장시간 운영 중 발생하는 각종 소프트웨어/OS 에러를 자동 복구합니다.

내부 자가 복구 (Watchdog): 유니티 내부 
프리징
 감지 시 강제로 Play() 재호출. 

파일 유실 등 치명적 오류 시 화면을 Index = 0 (첫 미디어)으로 리셋 
( 로그
 기록).

백그라운드 로깅 (Logger): 화면 에러 팝업 노출 없이 
Logger.cs
를 통해 지정된 폴더에 텍스트 파일로 에러 내역 누적 기록.
